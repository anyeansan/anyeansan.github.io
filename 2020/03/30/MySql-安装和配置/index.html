2.3 修改用户密码

  1. mysqladmin -u用户名 -p[旧密码] password 新密码
    • 不需登录
    • 旧密码可以写在命令中，也可以回车后输入
  2. alter user '用户名' @'localhost' identified by '新密码'
    • 需要登录
  3. 忘记 root 密码
    1. 关闭数据库
    2. 维护模式启动数据库
      • mysqld_safe --skip-grant-tables --skip-networking &
    3. 登录并修改密码
      1. alter user 'root' @'localhost' identified by '新密码'
      2. flush privileges
        • 如果修改密码报错，则刷新权限

3.2 客户端

1 MySQL自带的命令行工具 mysql.exe

1 连接方式

  1. TCP/IP 方式（远程 本地）
    • mysql -uuser -p[password] [-h Host] [-Pport]
  2. Socket 方式（本地）
    • mysql -uuser -p[password] [-S /tmp/mysql.sock]

2 myql 命令

  • mysql [OPTION]
    • OPTION
      • -u
      • -p
      • -h
      • -P
      • -S
      • -e "sql"
        • 免交互执行命令（即不连接数据库）
      • < 脚本路径
        • 导入 sql 脚本
      • -prompt 提示符
        • 定义 mysql 提示符，默认为 mysql>
        • \D：完整日期
        • \d：当前数据库
        • \h：服务器名
        • \u：当前用户

四 MySQL 体系结构

4.1 mysql 实例

  1. mysqld（老板）
  2. master thread（经理）
  3. other thread（员工）
  4. 预分配内存（办公区）

4.2 mysqld 程序结构

1 连接层

  1. 提供连接协议：TCP/IP 、SOCKET
  2. 提供验证：用户、密码、IP、Port、SOCKET …
  3. 提供专用连接线程：接收用户SQL，返回结果
    • show processlist
      • 查看连接线程

2 SQL 层

  1. 接收上层传送的SQL语句
  2. 语法验证模块：验证语句语法,是否满足SQL_MODE
  3. 语义检查：判断SQL语句的类型（DQL、DML …）
  4. 权限检查：用户对库表有没有权限
  5. 解析器：语句执行前进行预处理，生成解析树(执行计划)，说白了就是生成多种执行方案
  6. 优化器：根据解析器得出的多种执行计划，进行判断，选择最优的执行计划
    • 代价模型：资源（CPU IO MEM）的耗损评估性能好坏
  7. 执行器：根据最优执行计划，执行SQL语句，产生执行结果（在磁盘上）
  8. 提供查询缓存（默认没开启）：会使用redis tair替代查询缓存功能
  9. 提供日志记录（默认没开启）：二进制日志 binlog 、审计日志、通用日志 …

3 存储引擎层

  1. 根据SQL层执行的结果，从磁盘上拿数据
  2. 将16进制的磁盘数据，返回给 SQL 层结构化成表
  3. 由连接层的专用线程返回给用户

4.3 MySQL 逻辑存储结构

  1. 库（database）
    1. 库名
    2. 库属性（字符集，校对集(排序规则)）
  2. 表（table）
    1. 表名
    2. 表属性（存储引擎，字符集，校对集(排序规则)）
  3. 列（column）/ 字段（field）
    1. 列名
    2. 列属性（数据类型，约束，…）
  4. 行（row）/ 记录（record）

4.4 MySQL 物理存储结构

  1. 库（目录）
  2. 表
    1. MyISAM
      • .frm ：存储表结构（列名，列属性）
      • .MYD ：存储数据记录
      • .MYI ：存储索引
    2. InnoDB
      • .frm ：存储表结构（列名，列属性）
      • .ibd ：存储数据记录和索引
      • .opt ：存储库属性
      • ibdata1 ：数据字段信息
  3. InnoDB，一般情况下（非分区表）
    1. 段：一个表就是一个段，一个段由多个区构成
    2. 区：一个区由64个连续的页构造，1M 大小（ 64*16=1024 ）
    3. 页：一个页大小为 16 kb

五 用户 & 权限

5.1 用户

1 用户定义

  • user@'Host'
    • Host（白名单）
      • localhost
      • 127.0.0.1
      • %
        • 任意网段
      • 10.0.0.%
        • 10.0.0.0~10.0.0.255
      • 10.0.0.5%
        • 10.0.0.50~10.0.0.59
      • 10.0.0.0/255.255.255.0

2 用户管理

1 查询用户

  • select user,host,authentication_string from mysql.user
    • 查询用户名，主机，密码
    • 用户信息存储在数据库 mysql 中 的 user 表

2 创建用户

  • create user user@'Host' [identified by 'password']

3 删除用户

  • drop user user@'Host'

5.2 权限

1 权限列表

  1. ALL privileges
    • 所有权限
  2. with grant option
    • 可以给别的用户授权，超级管理员才具备

2 权限管理

1 查看权限

  • show grants for user@'Host'

2 授权

  • grant 权限 on 目标 to 用户 [identified by 'password'] [with grant option]
    • 权限
      1. ALL privileges
      2. 权限列表，以逗号分隔
    • 目标
      1. *.*
        • 所有数据库的所有表
      2. database.*
        • 指定数据库的所有表
      3. database.table
        • 指定数据库的指定表

3 回收权限

  • revoke 权限 on 目标 from user@'Host'
