Linux 综合架构

✦ 2020-03-29 ☛ Linux

一 中小规模网站综合架构组成

1.1 前端

  1. 防火墙
    • 访问策略控制
  2. 负载均衡服务器
    • 对访问请求进行调度处理
  3. web服务器
    • 处理用户请求

1.2 后端

  1. 数据库服务器
    • 存储字符数据，如订单…
  2. 存储服务器
    • 存储图片 音频 视频 附件都数据
  3. 备份服务器
    • 存储网站所有服务器重要数据
  4. 缓存服务器
    • 将常用数据存储到内存中，减轻服务器压力，提升用户体验

1.3 扩展

  1. 批量管理服务器
    • 批量管理多台服务器主机
  2. VPN服务器
    • 内部员工访问架构 PPTP vpn
  3. 审计服务器（跳板机）
    • 内部员工操作管理架构 jumpserver
  4. 监控服务器
    • 综合架构服务器的监控 zabbix
  5. 网站架构单点问题（高可用服务）
    1. 负载均衡服务器 keepalived
    2. 数据库 mha
    3. 存储服务器 分布式存储
    4. 备份服务 rsync 云存储
    5. 缓存服务 集群/哨兵模式

二 备份

2.1 备份策略

  1. 全量备份
    • 完整备份所有数据
  2. 增量备份
    • 做一次完全备份，以后每次只备份新增的数据
    • 例
      • 第一天完全备份
      • 第二天备份第二天新增数据
      • 第三天备份第三天新增数据
      • 第四天备份第四天新增数据
  3. 差异备份
    • 做一次完全备份，以后每次只备份和第一次完全备份不同的数据
    • 例
      • 第一天完全备份
      • 第二天备份第二天新增数据
      • 第三天备份第二天和第三天新增数据（和第一次比较）
      • 第四天备份第二天第三天和第四天新增数据（和第一次比较）

2.2 备份命令

  1. 本地备份
    • cp
  2. 远程备份
    • scp -rp
      • -r
        • 递归
      • -p
        • 保持文件属性

2.3 rsync

1 概述

  1. Rsync是一款开源的、快速的、多功能的、可实现全量及增量的本地或远程数据同步备份的优秀工具
  2. 安装 rsync 软件包

2 本地备份

  • rsync [OPTION] SRC DEST
    • 等价于 cp 命令
  • rsync FILE
    • 等价于 ls 命令

3 远程备份

1 Push

  • rsync [OPTION] SRC [USER@]HOST:DEST
    • 等价于 scp 命令
    • OPTION
      • -a
        • 命令的归档参数，等价于 -rtopgDl
      • -v
        • 显示详细的传输信息
      • -z
        • 传输时进行压缩以提高效率
      • -r
        • 递归
      • -t
        • 保持文件最后修改时间不变
      • -o
        • 保持文件属主信息不变
      • -p
        • 保持文件权限信息不变
      • -g
        • 保持文件属组信息不变
      • -D
        • 保持设备文件信息不变
      • -P
        • 显示数据传输的进度信息
      • -l
        • 保留软连接信息
      • -L
        • 保留软连接指向目标文件的数据
      • --exclude=PATTERN
        • 排除指定数据不被传输
      • --exclude-from=file
        • 排除指定数据不被传输(批量排除)
      • --bwlimit=RATE
        • 显示传输的速率
      • --delete
        • 让目标和源数据保持一致，即会删除目标中源没有的数据
    • USER@
      • 以什么用户身份推送数据（默认以当前用户）

2 Pull

  • rsync [OPTION] [USER@]HOST:SRC [DEST]
    • USER@
      • 以什么用户身份拉取数据（默认以当前用户）

4 守护进程方式（一直在后台持续的运行）

1 rsync 守护进程部署流程

  1. 安装
    • rpm -qa rsync
    • yum install -y rsync
  2. 配置文件 /etc/rsyncd.conf
  3. 创建rsync服务的虚拟用户
  4. 创建rsync服务密码认证文件（权限 600）
  5. 创建备份目录并修改属主属组信息
  6. 启动服务 systemctl start rsyncd

2 客户端免密码配置

  1. 创建一个密码文件（权限 600），存储密码
  2. 执行命令时，在命令后添加 --password-file =密码文件路径

3 Push

  • rsync [OPTION] SRC [USER@]HOST::DEST
    • OPTION
      • 常用 -avz
    • USER@
      • 指定认证用户信息
    • DEST
      • 指定备份服务器的模块

4 Pull

  • rsync [OPTION] [USER@]HOST::SRC [DEST]

5 rsync 原理

  1. 客户端当前登录用户通过命令向服务端发送推送请求
  2. 服务端判断客户端命令中认证用户是否存在配置 /etc/rsyncd.conf 中
  3. 服务端判断客户端认证用户密码是否正确 /etc/rsync.password
  4. 开始传输后，无论客户端发送请求的是root还是普通用户，到服务端都是通过配置的虚拟管理用户来备份。也就是说，传输的文件属主和属组在服务端会改为虚拟管理用户。
    1. 假如文件在客户端属主和属组是 root，服务端虚拟管理用户是 xiaoqiang，即在传输文件时需要把文件的属主和属组改为xiaoqiang，权限不足。
    2. fake super 配置会将虚拟管理用户伪装成root
